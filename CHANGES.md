How the consequence loop works end-to-end:
1. At generation time Claude defines 3-5 named flags (e.g. `tamed_dragon`, `helped_villager`, `chose_treasure`), 1-2 mid-tree choice scenes (`is_choice_scene: true` + 2 `choice_options`), 1-3 scenes with `flag_set` (typically side quests with pickups), 1-3 scenes with `narration_variants`, up to 3 with `prop_overrides`, and a top-level `endings` ranked list with a fallback `{}` entry as last.
2. Strict parser rejects: undefined flag refs in any of `flag_set` / `sets_flag` / `requires` keys / `narration_variants.when` / `prop_overrides.when` / `endings.requires`; choice scenes without exactly 2 options; prop_overrides on the starting scene; fewer than 2 endings; missing fallback as last.
3. At play time PlayClient owns `flags: Record<string, boolean>`, persisted per world in sessionStorage. StoryPlayer pulls flags + onSetFlag from props, resolves narration + props per scene via `resolveScene`, fires `flag_set` once on each scene's first exit, fires the chosen flag on ChoiceMoment activation. Choice scenes route via `goes_to`; the kid can never see the normal interactable layout for those scenes.
4. At finale: secret reroute (visit-all or all-pickups) wins highest priority and lands on `secret_ending`. Otherwise `selectEnding(tree.endings, flags)` resolves to the winning regular ending scene; if it differs from where the kid was routed, StoryPlayer redirects to it and lets the useEffect re-fire so the right narration shows. Then `world_completed`, `world_completed_with_ending` (only for non-secret), and the existing completion flow run.
5. RealmCard appends a flag-derived suffix to the title for the current playthrough. The suffix lookup is intentionally narrow: direct id matches first (e.g. `tamed_dragon` → "the Dragon Tamer"), then substring rules covering ~10 common motifs. Realms whose flag ids don't match anything just show the title with no suffix, which is fine.
6. The `world_endings` table accumulates per (user, world, ending) over time, so the kid can replay a realm twice via "Play again" or by re-entering it from /profile and unlock Two Endings; a third distinct ending unlocks All Three Endings.
